# Releasing

This project uses [`cargo-release`](https://github.com/crate-ci/cargo-release)
to bump the version, create a `v*` tag, and push it. The pushed tag triggers
[`.github/workflows/release.yml`](.github/workflows/release.yml) (generated by
`cargo-dist`), which cross-compiles the binary and creates a GitHub Release.

## Prerequisites

```sh
mise install   # installs Rust and cargo-release per mise.toml
```

You must be on `main`, with a clean working tree.

## Cutting a release

| Change                              | Command              | Example          |
| ----------------------------------- | -------------------- | ---------------- |
| Bug fix                             | `make release-patch` | `0.1.0 → 0.1.1` |
| Backwards-compatible feature        | `make release-minor` | `0.1.0 → 0.2.0` |
| Breaking change                     | `make release-major` | `0.1.0 → 1.0.0` |

Each command does the same five things:

1. bumps `version` in `Cargo.toml` and `Cargo.lock`
2. creates a release commit
3. creates a tag like `v0.1.1`
4. pushes the commit and tag to `origin`
5. exits — the rest is GitHub Actions

To preview without doing any of the above:

```sh
make release-dry
```

(Plain `cargo release patch` without `--execute` always dry-runs.)

## What happens after the tag is pushed

`release.yml` runs on every `v*` tag and:

- Builds for macOS (x86_64 + ARM64), Linux (x86_64 + ARM64), and Windows (x86_64).
- Computes archive checksums.
- Creates a GitHub Release with all artifacts attached.

Watch progress at <https://github.com/kukv/treedoc/actions/workflows/release.yml>.

If the workflow fails after the tag is already on origin, fix the issue and
delete + re-push the tag to retrigger:

```sh
git tag -d v0.1.1
git push origin :refs/tags/v0.1.1
git tag v0.1.1
git push origin v0.1.1
```

## Pre-releases

For `0.2.0-rc.0` style pre-releases:

```sh
mise exec -- cargo release rc --execute --no-confirm       # 0.1.0 -> 0.1.1-rc.0
mise exec -- cargo release rc --execute --no-confirm       # 0.1.1-rc.0 -> 0.1.1-rc.1
mise exec -- cargo release release --execute --no-confirm  # 0.1.1-rc.1 -> 0.1.1
```

`alpha` and `beta` work the same way.

## Specific version

```sh
mise exec -- cargo release 0.5.0 --execute --no-confirm
```

## Configuration

- `release.toml` controls cargo-release behavior (tag format, push, publish).
- `dist-workspace.toml` controls cargo-dist (targets, installers).
- `Cargo.toml` `[profile.dist]` controls how release binaries are compiled.
